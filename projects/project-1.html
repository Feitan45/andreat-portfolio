Project Overview

TLDR

Problem

  • Caregiving documentation is inconsistent and disjointed, families, caregivers, and clinicians lack shared visibility.
  • Early health changes often go unnoticed, leading to unmanaged symptoms and preventable hospitalizations.

Solution

  • AI-powered caregiving documentation system for home care
  • Design AI insights workflow
  • Improve detail in documentation
  • Simple, easy to navigate interface
  • Validated through usability feedback

Research

How we translated user needs into design direction.

Methods

  • User interviews at UC Berkeley senior center (n=8)
  • Market reserch: current tools are fragmented and documented "as needed", not daily
  • Competitors: Caring Village, Vesta Healthcare, Amada Healthcare

Pain Points

  • Documentation is long, disorganized


  • Caregivers struggle to remember daily responsibilities and rely on cluttered organization ecosystem

  • Users find it most difficult to identify mediication types and remember intake times.

  • Users struggle to keep track of important health documents, updates, and changes in health.

Needs

  • Quick capture for busy caregivers (voice input, templates)

  • Reduce cognitive load for task tracking



  • Simplified UI for users with varying digital literacy



  • Place to store important documents

Key user quote

“I use reminders for drugs and medications, notes for any side effects, and text messages for the family… it is hard to keep track of when things happened and even what happened that day… There is just too much information.”

Synthesis

Convert raw inputs into clear product direction: themes, jobs-to-be-done, and prioritization.

Themes

  • Fragmented tools: tasks, meds, notes spread across apps and chats
  • Cognitive Overload: caregiving is exhausting, pre-existing tools add burden on top.
  • No centralized health narrative or trend-tracking: hard to prep for doctor visits;

Design Principles

  • Voice note-taking as an option
  • Simple, straightforward UI
  • AI surface insights and pattern trends to be reviewed with healthcare professionals.

Ideation / Exploration

Highlight divergent thinking + why you converged on the final direction.

Explored Concepts

  • Voice journal with live transcript + edit
  • Structured “prompt cards” for care events
  • Insights dashboard for trend detection

Decision Rationale

  • Minimized steps for capture
  • Improved user confidence via visual feedback
  • Balanced structure + flexibility

Information Architecture

System thinking: navigation, content model, and user flows.

Primary Navigation

  • Home / Today
  • Journal
  • Insights
  • Tasks / Care plan

Content Structure

  • Nutrition
  • Medication
  • Personal Care
  • Activities
  • Sleep and Rest
  • Assessment

Core Flows

  • Start voice entry → transcript → edit → save
  • Review entries → filter/search → share
  • Insights → drill down to supporting entries

Final Screens

Present your strongest screens with short captions explaining the “why.” Keep it visual and scannable.

Usability Testing

conducted A/B testing on voice journal screens.

Test Plan

  • Participants: n= 10 caregivers
  • Prototype: mid/high-fi interactive
  • Tasks: find entry, log entry, edit entry, confirm,

Key Iterations

  • improved visual hierarchy and prompting
  • reduce work load of task documentation
  • improved level of details in documenting caregiving tasks

Reflection

What went well

  • Aligned stakeholders around clear user pain points
  • Designed for real constraints (hands-busy, time-poor)
  • Delivered implementable specs for engineering

What I’d improve next

  • Expand longitudinal insights + personalization
  • Support collaborative care team roles
  • Accessibility audit + localization considerations
